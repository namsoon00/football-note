# ADR-0001: 훈련일지 기반 로컬 운세 기능

- 상태: Accepted
- 날짜: 2026-03-04
- 작성자: Codex / Project Team

## 1. 배경 (Context)

훈련일지 작성 직후 사용자 리텐션을 높이고, 기록 행동에 즉시 보상을 주기 위해 `운세형 피드백` 기능이 필요했다.

요구사항:
- 프로필 정보 + 훈련 정보 + 날짜를 혼합해 운세 문구 생성
- 저장 시점에 즉시 결과 확인 가능
- 네트워크 의존 없이 동작
- 백업/복원 시 운세 데이터도 함께 이전

## 2. 결정 (Decision)

외부 API 호출 없이 동작하는 `LocalFortuneService` 기반 규칙 엔진을 채택한다.

세부 결정:
- 운세 생성 방식: 완전 로컬 규칙 + 시드 기반 문구 다양화
- 입력 신호:
  - 날짜: 연/월/일, 요일, 계절
  - 프로필: 이름, 생년월일(나이), 축구 시작일(구력), 별자리
  - 훈련: 시간, 강도, 컨디션, 부상/통증, 목표/메모, 프로그램, 리프팅 분포
  - 추세: 최근 14일 기록, 연속 훈련(streak)
- 출력:
  - 운세 본문 (`fortuneComment`)
  - 추천 훈련 문장 (`fortuneRecommendation`)
  - 추천 프로그램 (`fortuneRecommendedProgram`)
- UI:
  - 일지 작성 화면에 “오늘의 운세” 카드 노출
  - 간단/상세 토글 제공
  - 운세 이력(최근 7개) 바텀시트 제공
  - “추천 훈련 적용” 버튼으로 프로그램 자동 반영

## 3. 데이터 모델 변경

`TrainingEntry`에 필드 추가:
- `fortuneComment`
- `fortuneRecommendation`
- `fortuneRecommendedProgram`

또한 `drive_backup_service` 백업/복원 맵에 위 3개 필드를 포함한다.

## 4. 고려한 대안 (Alternatives Considered)

### A. 외부 운세 API 연동
- 장점: 데이터 소스 다양성 확보 가능
- 단점: 네트워크 실패/지연, 비용, 개인정보 이슈, 오프라인 불가
- 결론: 1차 릴리즈에는 부적합

### B. LLM 실시간 생성
- 장점: 문장 다양성과 자연스러움
- 단점: 비용/지연/안정성/개인정보 관리 부담
- 결론: 로컬 규칙형 안정화 후 2차 단계로 검토

## 5. 결과 및 영향 (Consequences)

긍정:
- 오프라인에서도 즉시 응답 가능
- 예측 가능한 비용(0원)과 안정성
- 백업/복원 일관성 확보

부정/제약:
- “정확한 운세”를 보장하는 기능이 아니라 `규칙 기반 동기부여 피드백` 성격
- 도메인 확장 시 규칙 유지보수 비용 증가

## 6. 정확성/표현 원칙

- 본 기능은 과학적 예측이 아닌 `게임화/동기부여 UX`로 취급한다.
- 사용자 신뢰를 위해 분석 근거(입력 신호)를 문구에 포함한다.
- 건강/부상 신호는 보수적으로 해석하여 과부하 권장을 피한다.

## 7. 운영 규칙

- 히스토리 저장: 최근 7개
- 저장 시점 자동 생성: Add/Update 모두 적용
- 추천 프로그램 자동 적용은 사용자 명시 동작(버튼 탭)으로만 실행

## 8. 후속 작업 (Follow-ups)

- `운세 기능 ON/OFF` 설정 토글 추가
- 연령대별 코칭 규칙 테이블 분리(설정 가능)
- A/B 테스트: 운세 카드 노출 방식(간단/상세 기본값) 비교
- 필요 시 외부 데이터(RAG/지식베이스) 연동 검토
